## Component Structure Pattern
```
packages/ui/src/components/ComponentName/
├── ComponentName.tsx         # Implementation with exported interface
├── ComponentName.stories.tsx # Stories using CSF3 format
└── index.ts                  # Barrel: export component + all types
```

## Storybook Setup
- Config: `packages/ui/.storybook/` (main.ts, preview.ts, ThemeDecorator.tsx)
- ThemeDecorator wraps every story with theme-aware background + font
- Themes: 'alpha' (green) and 'beta' (blue) switchable from toolbar
- Story discovery: `src/**/*.stories.@(ts|tsx)`
- Start: `pnpm exec storybook dev -p 6006 --ci` (use --ci to avoid interactive prompts)

## Inline Styles Pattern
All components use inline `style={{}}` objects — no CSS files, no class names.
Animation keyframes are injected via `<style>{ANIMATION_STYLES}</style>` as JSX sibling.

## Barrel Export Pattern
- `packages/ui/src/index.ts` — root barrel, add every new component here
- `packages/ui/src/components/OddsButton/index.ts` — re-export component + ALL types

## Type Export Convention
Export the Props interface AND any union type aliases (Variant, Size, State, etc.) from both
the component's index.ts and the root src/index.ts.

## Sole Permitted Hardcoded Color — Extended Note
`#fff` is allowed on ANY dark-background badge or button (e.g. live badge on `--color-error`,
selected OddsButton on `--color-primary`). Must always be accompanied by an inline comment.
